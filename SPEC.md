# 챗씨부동산 — 기획 스펙 (빌드 전 저장용)

> 내부 폴더/manifest name 제안: `chatssi-realestate`
> 챗씨부인 시리즈 네이밍 (챗씨부인 ↔ 챗씨부동산)

## 0. 개요
SillyTavern 확장. 두 개의 탭으로 구성됨.
- **탭1**: 거주지(주소 + 집 구조 + 부동산 정보) 생성
- **탭2**: 소지품 인벤토리 (공간별 아이템 풀, 포인트 해금)

## 1. 의존성
- **챗씨부인**: 불필요. 캐릭터시트/페르소나/로어북은 SillyTavern 컨텍스트에서 직접 읽음.
- **챗틀로얄**: 선택적 연동. 설치되어 있으면 포인트 합산, 없어도 독립 작동 (graceful fallback).

## 2. 탭1 — 거주지 생성

### 2.1 입력 영역
- 세계관 카테고리 칩: `자동감지` / `현실` / `판타지` / `시대극` / `메이저IP`
  - 기본값 자동감지 → 채팅방 정보로 시스템이 판단
  - 유저가 직접 지정하면 우선 적용
- 참고 텍스트 입력칸 (자유, 선택사항 — 하나의 칸에 세계관과 위치를 함께 적을 수 있음)
  - 예: `뉴욕 맨하탄`, `조선 한성`, `영국 헤리티지 시골마을`, `해리포터-런던`, `콜옵-모던워페어-뉴욕`
  - 비워두면 채팅방 데이터 기반 완전 자동 생성
- "집 생성하기" 버튼

### 2.2 메이저IP 처리 규칙
- 하위 작품 목록을 사전 정의하지 않음 — AI가 자체 지식으로 추론
- 모호한 경우(예: "콜옵"처럼 서브시리즈가 많은 IP) 우선순위:
  1. 채팅방 컨텍스트(캐릭터시트/로어북)에 힌트가 있으면 그걸로 판별
  2. 유저가 텍스트 입력에 구체적으로 적은 경우 그걸 우선 적용
  3. 둘 다 없으면 기본값 사용 — **콜옵 기본값: 모던워페어 리부트**
- 키워드는 **부분 일치(퍼지 매칭)**로 인식. 전체 문자열이 정확히 일치하지 않아도 됨
  (예: "블옵", "블랙옵스", "콜 오브 듀티 블랙옵스" 전부 같은 세계관으로 인식)

### 2.3 출력 — 부동산 카드(Deed) 형식
**항목별로 칸을 나눠서 표시** (줄글 서술 금지, 부동산 매물 카드처럼):

| 항목 | 설명 |
|---|---|
| 거주형태 | 월세/매매 (국가별 명칭 차이 반영, 예: 한국 전세 포함) |
| 가격 | 해당 국가 통화 + 그 나라 실제 시세 기준 |
| 건물유형 | 아파트/단독주택/코티지/타운하우스 등 (세계관별 어휘 치환 가능) |
| 방 개수 · 화장실 개수 | |
| 구조스타일 | 오픈플랜 / 유럽식 분리형 등 |
| 마당 유무 | |
| 차고 유무 | |
| 위치 | 실제 지역명 (동네 단위까지) |
| **주소** | 구체적 번지/좌표 — 실제 좌표 기반으로 지도에 찍히게 함. 정확한 건물 동일성은 허구 가능 (개인 사용 목적, 우연 일치 문제 없음) — **※ 목업에서 빠졌던 항목, 빌드시 반드시 포함** |
| 입주/구매 시점 | |
| 인테리어 스타일 | |
| 리모델링 여부/시기 | |
| 집 이야기 | 서술형 TMI 1단락 (예: 구옥 매입 후 리모델링한 사연) |
| 상태 | 자가거주중 / 임대중 등 |

### 2.4 부록 섹션
- 추가 보유 자산 (세계 각지 분산 부동산 등 — 캐릭터 재산 수준에 따라 분량 결정)
- 기타 TMI

### 2.5 거주 이력
- 이전 정보는 **삭제 없이 누적 보관** (탭1은 히스토리 기록형 — 탭2의 풀 순환 방식과 다름)
- 현재 카드만 메인에 노출, 과거 기록은 아코디언/서브리스트로 펼쳐보기

### 2.6 이사가기 버튼 (수동, 자동감지 없음)
- 자동연동 토글 제거 — 자동 감지 로직 자체를 두지 않음
- **🚚 이사가기 버튼**만 존재. 누르면 SillyTavern이 실제로 LLM에 보내는 **최종 조립 프롬프트 전체**를 한 번에 읽어옴
  (`generateQuietPrompt({quietToLoud:true})` 류 방식 — 챗씨부인에서 이미 쓰던 기법과 동일.
  캐릭터시트/페르소나/활성화된 로어북/최근 챗 히스토리가 이미 다 합쳐진 상태라 따로 긁어올 필요 없음)
- 동작은 "집 생성하기"와 동일하되, 기존 카드는 삭제하지 않고 거주 이력으로 이동시킨 뒤 새로 생성
- **정보블록 필터링 필수**: 다른 확장이 주입한 상태창/정보블록(고정 태그·구분자로 감싸진 블록)이
  같은 프롬프트 안에 섞여 있을 수 있음 → 생성 프롬프트에 "해당 형식의 텍스트는 무시하고 참고하지 말 것"
  가드 문구 포함, 또는 읽는 시점에 정규식으로 사전 제거. 실제 빌드 시 각 정보블록 확장이 쓰는
  구분자 패턴 확인 필요
- 탭2와는 무관, 탭1에만 적용

### 2.7 로어북용 복사 버튼
- 카드 하단에 "📋 로어북용 복사" 버튼
- 카드의 모든 항목 데이터를 **카테고리별로 묶은 줄글**로 변환해서 클립보드에 복사
  (예: [거주지] / [구조] / [인테리어] / [입주 이력] / [기타] 섹션으로 나눠진 문단)
- 그대로 SillyTavern 로어북 엔트리에 붙여넣기 가능한 형태
- 복사 완료 시 짧은 토스트 메시지로 피드백

## 3. 탭2 — 소지품 인벤토리

### 3.1 고정 공간 — 기능 카테고리는 불변, 표현은 시대별 가변

| 기능 카테고리 (불변) | 현대 예시 | 시대극(조선) 예시 | 판타지 예시 |
|---|---|---|---|
| 음식보관 | 주방 (팬트리/냉장고 서브뷰) | 주방 (찬장/장독간) | 주방 (식료품창고/마법냉기함) |
| 이동수단보관 | 차고 (차/오토바이/자전거) | 마구간(말) 또는 없음 | 차고 (그리폰 우리/마차) |
| 위생공간 | 욕실 | 측간/목욕간 | 욕실 |
| 수면공간 | 침실 (드레스룸: 옷/시계/주얼리/신발 포함) | 침실(안방) | 침실 |
| 지식/취미공간 | 서재 | 서재(사랑방) | 서재(마법서고) |
| 기타보관 | 창고 | 창고(곡식창고) | 창고 |

- UI 골격(6칸 그리드)은 항상 동일, 시대 파싱 결과에 따라 라벨/아이콘/서브뷰 존재여부가 결정됨
- 그 시대에 의미 없는 슬롯(예: 캐릭터가 말이 없는 조선시대의 마구간)은 빈 상태/대체 문구로 처리
- 캐릭터 재산수준 낮으면 일부 슬롯 비거나 소박하게 (차고 없음 등)
- 부자는 대표 집 외 자산은 탭1 부록의 "세계 각지 분산" 텍스트로 번외 처리

### 3.2 음식보관 공간(주방) 세부
- 팬트리/냉장고는 **별도 풀스크린 서브뷰**로 전환 (그리드/포인트바 숨김, 뒤로가기 버튼 + 팬트리·냉장고 토글)
- 일반 식료품: 무잠금, 단순 리스트(이름 + 수량), 포인트 무관
- 고가/특별 식료품(수입와인, 빈티지 위스키 등 소수): 포인트로 해금
- 냉장고 개념 없는 시대(조선시대 등)는 해당 서브뷰 자체가 생성 안 되거나 다른 명칭(장독간 등)으로 대체

### 3.3 아이템 풀 시스템 (드레스룸/거실/서재/차고/창고 공통)
- 공간당 총 캡: **12개 슬롯** (3열 그리드에 꽉 채워지는 개수로 조정, 무한 생성 방지)
- 최초 생성: 채팅/로어북에서 확인된 아이템 + AI 추측 아이템을 섞어서 캡까지 채움, **랜덤 순서로 배치** (실제 반영분과 추측분 구분 안 보이게)
- 기본 1~2개 무료 해금, 나머지는 포인트로 해금 (개당 5~15P, 항목 등급별로 상이)
- **핀(고정) 기능**: 해금된 아이템 중 유저가 마음에 드는 것은 고정 → 교체 대상에서 영구 제외
- 풀이 캡까지 다 찼는데 새 스토리 아이템이 들어와야 할 때 → 고정 안 된 것 중 **가장 오래된 것부터 교체(재잠김)**
- 스토리 기반 추가분도 같은 캡/같은 풀에 포함 (별도 상한선 불필요 — 캡 하나로 통합 관리)

### 3.4 세계관별 아이템 성격
- 현실: 실제 브랜드명 + 실제 가격
- 판타지: 세계관 고유 아이템(마법봉/포션 등), "브랜드" 대신 장인/길드명으로 대체
- 시대극: 시대 고유 물품 + 시대 화폐단위
- 현대+판타지 혼합 세계관: 한 공간에 현실 브랜드와 세계관 아이템이 같이 등장 가능
- 완전 새 세계관(SF 등): 창작된 가상 브랜드명 사용

### 3.5 아이템 설명 (모달 팝업)
- 브랜드명 + 아이템명
- 가격 (현지화폐 또는 달러)
- TMI (선물/직접구매 등 사연 — 게임 아이템창 설명 톤)
- 핀(고정) 버튼

### 3.6 페르소나용 비밀 아이템
- 캐릭터가 **페르소나(유저 자신의 캐릭터)를 위해 몰래 준비해둔 물건**도 풀에 섞일 수 있음
  (예: 침실 서랍에 숨겨둔 반지, 아직 안 준 선물 등)
- 채팅 히스토리/관계 맥락(애정도, 최근 대화에서 선물 언급 등)을 읽고 추론해서 생성
  — 직접적인 언급 없어도 관계 분위기상 그럴듯하면 생성 가능
- TMI에 "아직 안 줬다", "줄 타이밍 보고 있다" 같은 비밀스러운 사연으로 표시
- 다른 일반 아이템과 같은 풀/캡 안에 포함, 별도 카테고리로 분리하지 않음 (서프라이즈 요소로 자연스럽게 섞여있는 느낌)

## 4. 포인트 시스템
- "챗씨부동산" 자체 **독립 포인트 시스템** 보유 → 챗틀로얄 없어도 정상 작동
- 신규 설치 시 기본 지급: **50P**
- 이후 **1시간당 1P** 자동 적립
  - 구현 방식: 마지막 적립 타임스탬프 저장 → ST 재접속 시 경과시간 계산해서 일괄 캐치업 적립 (`setInterval` 단독 방식은 ST 꺼져있으면 적립 안 되므로 부적합)
- 챗틀로얄 설치 시 → 그 포인트 데이터를 감지해서 **합산** (graceful fallback: 없으면 그냥 스킵, 에러 없이)

## 5. 기술 노트 (빌드시 참고 — 챗씨부인/챗틀로얄 실제 코드 기준으로 검증됨)
- **컨텍스트 접근**: `SillyTavern.getContext()`로 항상 접근 (import된 getContext 아님)
- **설정 저장**: `ctx.extensionSettings[MODULE_NAME]`에 저장, `ctx.saveSettingsDebounced()`로 저장
- **AI 호출**: `ctx.generateQuietPrompt({ quietPrompt, quietToLoud:true, skipWIAN:false })` 한 번 호출로
  캐릭터시트/페르소나/로어북/최근 챗 히스토리가 이미 자동으로 컨텍스트에 섞인 채 생성됨.
  컨텍스트를 별도로 "읽어와서" 텍스트로 프롬프트에 박아넣을 필요 없음 (이전 계획 수정됨).
  단, 이 방식은 ST의 "현재 연결"을 사용하며 별도 API 프로필 선택을 지원하지 않음 — 별도 프로필을
  쓰고 싶다면 `ctx.ConnectionManagerRequestService.sendRequest(profile.id, messages, maxTokens, options)`
  방식으로 가야 하지만, 이 경우 로어북/챗 컨텍스트 자동 포함이 안 되므로 직접 모아서 프롬프트에
  넣어줘야 함 (챗씨부인의 `callAI()` 두 갈래 패턴 참고)
- **정보블록 필터링**: 자동 포함되는 컨텍스트라 직접 정규식 제거는 어려움 → 프롬프트 내
  가드 문구(INFO_BLOCK_GUARD)로 "무시하라"고 명시하는 방식으로 대체
- **챗틀로얄 포인트 키 확인됨**: `ctx.extensionSettings['chatl_royal'].points` (모듈명 `chatl_royal`)
- **데이터 단위**: 챗(대화) 단위가 아니라 **캐릭터 단위**로 저장 (캐릭터당 하나의 거주지/소지품 데이터,
  여러 채팅방을 오가도 유지됨) — `ctx.characters[ctx.characterId].avatar`를 키로 사용
- **UI 마운트**: 외부 style.css 없이 인라인 스타일로 통일 (챗씨부인/챗틀로얄과 동일 컨벤션).
  `#extensions_settings`에는 간단한 설정 드로어만, 실제 인터랙티브 패널은 `document.body`에
  별도로 추가하는 드래그 가능한 플로팅 패널 (`makeDraggable` 패턴 재사용)
- **유틸리티**: `esc()` (HTML 이스케이프), `toastr.success/error/warning`, `Popup.show.confirm()` —
  전부 챗씨부인/챗틀로얄에서 검증된 패턴 그대로 재사용
- **설정 UI 안전장치**: "전체 초기화" 버튼은 `Popup.show.confirm()`으로 확인 후 처리
- 데이터 저장: `extension_settings[extensionName]` — 캐릭터별/채팅별로 최소 데이터만 저장 (생성된 집 정보, 아이템 풀, 타임스탬프), 전체 채팅로그 통째 복제 금지 (캐시 비대화 방지)
- AI 호출: Connection Manager `ConnectionManagerRequestService` 사용, 확장 설정에 프로필 선택 드롭다운 별도 제공
- 설정 UI에 "전체 초기화" 버튼 필요 (extension_settings 통째로 비우기)
- 캐릭터/채팅 삭제 시 관련 키 정리 훅 필요 (죽은 데이터 누적 방지)
- 챗틀로얄 연동: 해당 extension_settings 키 존재 여부 먼저 체크 후 합산하는 방식으로 구현
