# 챗씨부동산 — 프롬프트 초안 모음

> 실제 빌드시 `prompts.js`로 분리될 블록들. 지금은 초안 단계 — 톤/디테일은 빌드하면서 다듬을 것.
> CANON 스타일 컨벤션 참고: 지시문 위주, 예시 최소화, 모듈 단위 분리.

---

## PROMPT_LOREBOOK_EXPORT (로어북용 줄글 변환)

```
역할: 로어북 엔트리 작성기.
입력: 현재 거주지 카드 데이터(항목별 구조화 데이터).

작업: 카드의 모든 항목을 카테고리별로 묶어서 자연스러운 줄글 문단으로 변환하라.
  카테고리 구성(고정):
    [거주지] — 위치/주소/거주형태/가격/상태
    [구조] — 건물유형/방·화장실/구조스타일/마당/차고
    [인테리어] — 인테리어스타일/리모델링/집 이야기
    [입주 이력] — 입주시점 + 거주 이력 요약
    [기타] — 부록(추가자산/TMI)

문체: 캐릭터시트/로어북에 어울리는 3인칭 서술체. 표 형식이나 항목 나열 금지,
완전한 문장으로 자연스럽게 이어쓸 것. 과장된 수식어 자제, 정보 전달 위주.

출력 형식: 카테고리 라벨(`[거주지]` 등)을 헤더로 둔 일반 텍스트 (줄글)
```

## PROMPT_WORLD_CLASSIFY (세계관 분류)

```
역할: 캐릭터 데이터 분석가.
입력: 캐릭터시트, 페르소나, 로어북, (있다면) 유저가 적은 참고 텍스트.
작업: 아래 4개 카테고리 중 하나로 분류하라.
  - REALISTIC: 현실 세계, 실제 국가/도시 기반
  - FANTASY: 오리지널 판타지/이세계
  - HISTORICAL: 특정 시대극 (어느 시대/지역인지 구체적으로 명시)
  - MAJOR_IP: 기성 유명 작품 세계관 (작품명 추론해서 명시)

유저 참고 텍스트가 있다면 최우선 반영. 텍스트에 "세계관-위치" 형태(예: "해리포터-런던")가
있으면 세계관과 위치를 분리해서 인식하라.

MAJOR_IP인데 세부 시리즈가 여러 개로 갈리는 경우(예: 콜오브듀티):
  1순위 - 캐릭터데이터/로어북에 단서가 있으면 그걸로 판별
  2순위 - 유저 텍스트에 구체적 시리즈명이 있으면 그걸로 판별 (부분 일치 허용,
          "블옵"/"블랙옵스"/"콜 오브 듀티 블랙옵스" 전부 동일 인식)
  3순위 - 둘 다 없으면 기본값 사용 (콜오브듀티 → 모던워페어 리부트)

출력 형식: JSON
{ "category": "...", "subtype": "...", "location_hint": "..." }
```

## PROMPT_ADDRESS_GENERATE (거주지/주소 생성)

```
역할: 부동산 정보 생성기.
입력: PROMPT_WORLD_CLASSIFY 결과, 캐릭터 재산수준/직업/국가 추정치, 유저 참고 텍스트.

작업: 캐릭터의 거주지 정보를 아래 항목으로 생성하라. 줄글 금지, 항목별 단답 위주.
  - 거주형태 (월세/매매, 국가 관습 반영 — 한국이면 전세 옵션도 고려)
  - 가격 (해당 국가 통화, 그 지역 실제 시세 범위 안에서)
  - 건물유형
  - 방 개수 / 화장실 개수
  - 구조스타일 (오픈플랜 / 분리형 등)
  - 마당 유무
  - 차고 유무
  - 위치 (실제 지역명, 동네 단위)
  - 주소 (구체적 번지/좌표 — 동네 안에서 임의 좌표 선택, 정확한 건물 동일성은 신경쓰지 않음)
  - 입주/구매 시점
  - 인테리어 스타일
  - 리모델링 여부/시기
  - 집 이야기 (1단락, TMI 톤)
  - 상태

세계관이 REALISTIC이 아닌 경우, 위 항목의 "어휘"만 그 세계관에 맞게 치환하라
(가격 단위, 건물유형 명칭, 주소 표기법 등). 항목 구조 자체는 유지.

캐릭터 재산수준이 낮으면 차고/마당 등 일부 항목은 "없음"으로,
재산수준이 매우 높으면 부록에 추가 자산 1~2개 별도 생성.

출력 형식: JSON (위 항목별 key-value)
```

## PROMPT_HOUSE_MOVE (이사가기 — 수동 트리거, 자동감지 없음)

```
역할: 거주지 재생성기. (자동 감지 없음 — "이사가기" 버튼 클릭시에만 호출)
입력: SillyTavern이 LLM에 보내는 최종 조립 프롬프트 전체 (generateQuietPrompt류 방식으로
      획득 — 캐릭터시트/페르소나/활성화된 로어북/최근 챗 히스토리가 이미 합쳐진 상태),
      현재 저장된 거주지 카드.

⚠ 필터링 지시: 입력 프롬프트 안에 다른 확장이 주입한 상태창/정보블록(고정 태그나
구분자로 감싸진 텍스트, 예: [STATUS], <status> 등)이 포함되어 있을 수 있다.
해당 블록은 참고하지 말고 완전히 무시할 것. 그 블록의 형식, 수치, 표현을
출력에 절대 반영하지 말 것.

작업: 위 컨텍스트를 바탕으로 PROMPT_ADDRESS_GENERATE와 동일한 항목 구조로
새 거주지 카드를 생성하라. 기존 카드 데이터는 참고용으로만 사용하고
(같은 캐릭터의 톤/일관성 유지 목적), 그대로 복사하지 말 것.

출력 형식: JSON (PROMPT_ADDRESS_GENERATE와 동일한 key-value 구조)
+ 기존 카드는 호출부에서 거주 이력 리스트로 이동 처리 (이 프롬프트 책임 아님)
```

## PROMPT_ITEM_POOL_GENERATE (공간별 아이템 풀 생성)

```
역할: 소지품 인벤토리 생성기.
입력: SillyTavern이 LLM에 보내는 최종 조립 프롬프트 전체 (generateQuietPrompt류 방식으로
      획득 — 캐릭터시트/페르소나/활성화된 로어북/최근 챗 히스토리가 이미 합쳐진 상태,
      PROMPT_HOUSE_MOVE와 동일한 입력 소스), PROMPT_WORLD_CLASSIFY 결과, 대상 공간 1개
      (주방/거실/욕실/침실/서재/차고/창고 중 하나).

⚠ 필터링 지시: 입력 프롬프트 안에 다른 확장이 주입한 상태창/정보블록(고정 태그나
구분자로 감싸진 텍스트, 예: [STATUS], <status> 등)이 포함되어 있을 수 있다.
해당 블록은 참고하지 말고 완전히 무시할 것. 그 블록의 형식, 수치, 표현을
출력에 절대 반영하지 말 것.

작업:
1. 먼저 이 공간이 현재 세계관/시대에서 어떤 형태로 존재하는지 결정
   (예: 시대극에서 "차고"는 마구간이 되거나 존재하지 않을 수 있음 — 존재하지 않으면
   empty_reason만 반환하고 종료)
2. 존재한다면, 캐릭터 데이터에서 실제로 확인되는 아이템과 캐릭터 성격/재산수준 기반
   추측 아이템을 합쳐서 정확히 12개 슬롯을 채울 것
   - 실제 확인된 아이템과 추측 아이템을 구분 표시하지 말고 무작위 순서로 배치
   - 이 중 0~1개는 **페르소나(유저 캐릭터)를 위해 캐릭터가 몰래 준비해둔 물건**일 수 있음.
     채팅 히스토리/관계 맥락(애정도, 최근 대화에서 선물 언급, 관계 진행도 등)을 읽고
     그럴듯하면 생성할 것. 직접적인 언급이 없어도 관계 분위기상 자연스러우면 포함 가능.
     없을 것 같으면 생성하지 않아도 됨 (강제 아님).
3. 각 아이템은 아래 필드 포함:
   - emoji (대표 이모지 1개)
   - name (아이템명)
   - brand (브랜드/장인/길드명 — 세계관에 맞게)
   - price (현지화폐 또는 세계관 화폐 단위)
   - tmi (1~2문장, 게임 아이템창 설명 톤 — 선물/구매 이유 등. 페르소나용 비밀 아이템인 경우
     "아직 안 줬다", "줄 타이밍을 보고 있다" 같은 비밀스러운 사연으로 작성)
   - unlock_cost (5~15 중 하나, 기본 해금 슬롯 1~2개는 0)
4. 세계관별 아이템 성격 적용:
   - REALISTIC → 실제 브랜드/가격
   - FANTASY → 세계관 고유 아이템, 브랜드는 장인/길드명
   - HISTORICAL → 시대 물품 + 시대 화폐
   - 혼합형 → 현실 브랜드 + 세계관 아이템 혼재 가능
   - 신규 세계관(SF 등) → 창작 가상 브랜드

음식보관 공간(주방)의 경우 별도 처리:
   - 일반 식료품은 unlock_cost 없이 이름+수량만 (브랜드/TMI 불필요)
   - 소수의 고급/특별 식료품만 unlock_cost 부여

출력 형식: JSON 배열 (12개 항목, 또는 empty_reason)
```

## PROMPT_ITEM_POOL_REFRESH (스토리 반영 — 풀 교체)

```
역할: 인벤토리 업데이트기.
입력: 최근 채팅 메시지, 현재 아이템 풀(고정 여부 포함), 대상 공간.

작업: 최근 대화에서 캐릭터가 새로 얻은/구매한/받은 물건이 있는지 확인.
  있다면: 고정(pinned)되지 않은 아이템 중 가장 오래된 것을 선택해 교체 대상으로 지정,
          새 아이템 데이터(PROMPT_ITEM_POOL_GENERATE와 동일 필드) 생성.
  없다면: "no_new_item" 반환.

출력 형식: JSON
{ "new_item_detected": true/false, "replace_index": n | null, "new_item": {...} | null }
```
